# 3. UX Structure (Concrete)

## A. Main Surface = “Decision Feed”

Think:

- Twitter/X feed structure
    
- RSS freshness
    
- But every item is **typed + owned + auditable**
    

### Feed Item Anatomy

Each item is:

```
[ROLE] + [CLAIM] + [ARTIFACT] + [CONFIDENCE] + [NEXT ACTION]
```

Example:

- **Policy Analyst** → “$4.2B redirected to state voucher programs”
    
- Artifact → policy object
    
- Confidence → High
    
- Next → triggers Builder
    

This directly maps to your pipeline:

> Analyst → Builder → Reviewer → Operator → Truth-Teller

---

## B. Threaded Decision Chains (Critical)

Every feed item expands into a **thread**:

```
Policy Signal
  ↳ Distortion Model
    ↳ Validation Attack
      ↳ Execution Output
        ↳ Reality Outcome
```

This is the **visible system flow**.

No abstraction.  
No hiding.

# 4. Interaction Model (This is the real product)

Users don’t “search”.

They do three things:

---

## 1. Challenge

User clicks on any artifact and asks:

- “What assumption breaks this?”
    
- “Why is this real vs narrative?”
    

→ routes to **Reviewer / Truth-Teller only**

No other role can answer.

---

## 2. Extend

User asks:

- “What happens if this policy expands to X state?”
    

→ routes to **Builder**

---

## 3. Commit

User signals:

- “I want to act on this”
    

→ triggers:

- Operator surface
    
- execution pathways
    

---

This enforces:

> **role-constrained interaction, not free-form chat**

# 3 interaction modes

## 1. Feed Post

User posts to their own feed.

```md
"New EO dropped. What does this mean for education operators?"
```

The whole Synthetic Team is subscribed.

System decides who responds:

- Analyst extracts policy
    
- Builder models distortion
    
- Reviewer challenges it
    
- Operator suggests action
    
- Truth-Teller tracks outcomes
    
- Facilitator resolves conflict
    

This is the default mode.

---

## 2. @Role Mention

User directly invokes a role.

```md
@Reviewer break this claim.
@Builder model the upside.
@Analyst what actually changed?
```

Only that role has authority to respond.

This preserves role constraint.

---

## 3. @Team Mention

User invokes the full team.

```md
@Team assess this opportunity.
```

System runs a staged response:

```md
Analyst → Builder → Reviewer → Operator → Truth-Teller → Facilitator
```

This is heavier, slower, more formal.

Use for high-stakes decisions.

---

# Product Rule

The system can auto-route based on context, but routing must be visible.

Bad:

```md
ELaB says...
```

Good:

```md
Iris / Analyst responded because this is a policy-interpretation question.
```

No anonymous intelligence.
